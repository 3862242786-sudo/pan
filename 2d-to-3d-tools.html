首选：Depth Pro

次选：DUSt3R (CPU模式)
支持纯CPU运行，只需2张照片即可生成完整3D点云。处理时间约3-5秒（CPU），内存占用4-8GB。不需要相机参数，上手最简单。

🥉
备选：SHARP (CPU模式)
苹果最新的单图3D重建模型，CPU模式可运行但速度较慢（可能几分钟一张）。质量最好，但HD2500上可能需要降低输入图片分辨率来减少内存占用。
单图输入 CPU较慢 内存占用大
GitHub 示例
⚡ 低配电脑优化建议
  • 降低输入图片分辨率至 512x512 或更低，可大幅减少内存和计算量
  • 使用 DUSt3R 的 224px 模型替代 512px 模型，速度提升约4倍
  • 关闭不必要的后台程序，为模型推理留出更多内存
  • SHARP 可尝试 FP16 半精度模式，显存占用减半
  • 批量处理时建议一张一张来，避免同时加载多张图片
  • 如果内存不足（<8GB），优先选择 Depth Pro 或在线服务
  • Linux 系统通常比 Windows 内存管理更高效，可考虑双系统
  • 使用在线服务（Luma AI、Meshy）作为替代方案，零硬件要求
